# Changelog

## [1.0-002] - 2026-05-04

Initial release. All fixes are additive — the patch never modifies upstream Alba's files. Each fix below is keyed to the IDs used in the working notes (`alba_handoff.md` and `flour_eggs_milking_handoff.md`).

### Fixed

- **ALBA-1** — `ChangeCrops` no longer throws `[LUA ERROR] attempt to index a boolean value` on every game start / save load. Upstream's `local function ChangeCrops()` (`Code/AlbaVegetation.lua`) does `Resources.Potato.SpecificToRegions.Alba = true` and `Resources.Tomato.SpecificToRegions.Alba = true`. Vanilla `Resource.SpecificToRegions` defaults to `false` (boolean), so the index crashes and aborts the rest of `ChangeCrops`. Patch pre-inits Potato/Tomato `SpecificToRegions` to `{}` on `DataLoaded` / `ModsReloaded` / `NewGame` / `LoadGame` (each fires before `GameTimeStart` / `PostLoadGame`), and resets to `false` afterward in an additive `GameTimeStart` / `PostLoadGame` handler that runs after upstream's via dependency-driven load order.
- **STEAM-1** — Winemaking research now unlocks from CactusFruit harvests on Alba (Steam: Omega13 2025-09-30). Upstream's `ChangeCrops` adds `CactusFruit` to `Resources.CactusFruit.in_groups` and to `GroupResourceIds.Fruits` but doesn't sync `ResourceInGroupIds.CactusFruit` — a frozen cache built by `PreProcessResources` at `DataLoaded` time. `Player:UnlockResource(res)` walks `ResourceInGroupIds[res]` to recursively unlock parent groups, so the cascade never reaches `Fruits` and Winemaking stays gated. The patch's post-fix handler adds `Fruits` to the cache and calls `UIPlayer:UnlockResource("Fruits")` retroactively for existing saves where CactusFruit was already harvested.
- **STEAM-2** — `Tech.FieldBerrycone.Description` now references `res_tomatoes` instead of `res_heptagonian_honey` (Steam: sirbuche 2025-04-06). Both Description and TradeDescription advertise unlocking harvesting of Beefberries; the TradeDescription was already correct, but the Description showed the wrong icon. `ModItemChangeProp` on the same translation key (`T(885630558832, ...)`) keeps existing localizations valid.
- **ALBA-2** — Geysers no longer cluster around the negative-axis side of Energy Crystals. Upstream's `Alba_AddGeysers` uses `(-1 * geyser_random(2))` which evaluates to `{-1, 0}` (`geyser_random(2)` returns 0 or 1; multiplying by -1 yields 0 or -1). Corrected to `(-1 + 2 * geyser_random(2))` which evaluates to `{-1, +1}`. Patch overrides the global `Alba_AddGeysers` with the corrected version; upstream's `RegionDef` `ExecuteCode` call site picks up our version at game start.
- **ALBA-3** — `SavegameFixups.AddGeysers` no longer prompts the geyser-placement dialog on non-Alba save loads. Upstream registered the fixup without a region check, so any user with the mod installed who loaded a non-Alba save got prompted. Patch reassigns `SavegameFixups.AddGeysers` with `if not Game or Game.region ~= "Alba" then return end`.
- **Bonfire passive warmth** — hypothermic-but-already-relaxed survivors near a working bonfire now receive the cold-resistance buff (Steam: MommyDearest 2025-03-04). Upstream's `BonfireWarmUp` relaxation routine only fires when the survivor's Relaxation need is low enough to seek one out, stranding hypothermic survivors who already finished a relaxation activity. Patch adds a region-gated `MapGameTimeRepeat("Alba_BonfireWarmth", const.HourDuration / 4, ...)` thread that sweeps working bonfires every 15 in-game minutes and applies the existing upstream `BonfireColdResistance` happiness factor (StackLimit=1) to any human within `RadiationRange` (6000) carrying any `Hypothermia_*` health condition. Threshold-gated to hypothermic only — survivors who are merely cold but not yet hypothermic don't get the buff (preserves the user's "Alba winter difficulty is intentional" stance from the working notes).

### Deferred (not in this patch)

- **ALBA-4** — operator-precedence bug in upstream's `OnMsg.ModsReloaded` recapture guard (`AlbaVegetation.lua:307` pre-fix). The buggy line `if not Game.region == "Alba" then` parses as `if (not Game.region) == "Alba" then` (always false). Effect: `const_PlantTempDamagePerDay` never gets re-snapshotted from the live `const.PlantTempDamagePerDay`, so a session that switches from an Alba save to a non-Alba save leaves the cold-tolerance damage value at 15000 instead of vanilla. Not patchable from a dependent mod without shadowing upstream's local upvalue. Only affects mid-session region-switch flows.
- **ALBA-5** — dead-file cleanup of `Building/FarmField_SnowCactus.lua`. No realistic player impact; not a behavioral bug.
- **Particle texture path warnings** — `[Warning] Invalid location of particle texture Mod/LH_Alba/GeyserSteam.dds` and `GeyserWater.dds`. Emitted at preset-load time, before any `OnMsg` handler runs, so a patch mod fundamentally cannot silence them. Cosmetic only — the particle effects still play correctly.
